sourcing nba/wnba stats with go-etl package

all data is sourced from nba.com via an automated extract-transform-load (ETL) program I wrote in Go. the Go code sends http requests to stats.nba.com, transforms the returned json into Go structs matching the databaes design, & inserts the data into the appropriate database tables.
* Go ETL source code | github | nightly-bball-etl*
/-|-\
this site was originally powered by a MariaDB database with an ETL program written in Python. the Python program utilized the nba_api package to make http requests the stats.nba.com. I wanted to learn more of the lower-level concepts abstracted away by this package, so I decided to rewrite the entire ETL process in Go using (mostly) only the standard Go library. the documentation from the nba_api was incredibly helpful in this process.
* legacy python ETL | github | py-nba-mdb*

postgres database

the NBA/WNBA stats from jdeko.me/bball are served from a containerized postgres database server. the database was designed by myself and attempts to meet Codd's third normal form. the database build is fully automated in a shell script that builds & runs the postgres docker container, runs the Go ETL process, and executes several stored procedures. all SQL (DDL) statements to build the database are executed when the container is built. once the docker container is running and the database finishes building, the Go ETL process is used to source & insert data for every NBA/WNBA regular season/playoff game since 1970. the data is inserted into intake tables, which match the structure of the json from stats.nba.com exactly. after the Go program completes, a number of plpgsql stored procedures/functions are executed to insert the data from the intake tables into the 'lg', 'stats', and 'api' schemas of the postgres databse. these schemas make up the core of the normalized/relational design
